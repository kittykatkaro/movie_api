API Documentation

1. Return a list of ALL movies

Method: GET

URL: /movies

Description: This endpoint returns a list of all movies in JSON format.

Response:

        [
            {
                "title": "Movie Title",
                "description": "Movie description",
                "genre": { "name": "Genre Name", "description": "Genre description" },
                "director": { "name": "Director Name", "bio": "Director bio", "birth_year": 1970, "death_year": 2020 },
                "imageURL": "http://example.com/image.jpg",
                "featured": true
            },
            ...
        ]

2. Return data about a single movie by title

Method: GET

URL: /movies/:title

Description: This endpoint returns detailed information about a specific movie based on its title.

Request Params:

{ "title": "Movie Title" }

Response:

        {
            "title": "Movie Title",
            "description": "Movie description",
            "genre": { "name": "Genre Name", "description": "Genre description" },
            "director": { "name": "Director Name", "bio": "Director bio", "birth_year": 1970, "death_year": 2020 },
            "imageURL": "http://example.com/image.jpg",
            "featured": true
        }

3. Return data about a genre by name/title

Method: GET

URL: /genres/:name

Description: This endpoint returns the description of a genre by its name.

Request Params:

{ "name": "Genre Name" }

Response:

        {
            "name": "Thriller",
            "description": "Genre description"
        }

4. Return data about a director by name

Method: GET

URL: /directors/:name

Description: This endpoint returns the bio, birth year, and death year of a director by their name.

Request Params:

{ "name": "Director Name" }

Response:

        {
            "name": "Director Name",
            "bio": "Director bio",
            "birth_year": 1970,
            "death_year": 2020
        }

5. Allow new users to register

Method: POST

URL: /users

Description: This endpoint allows a new user to register by providing their username, password, email, and date of birth.

Request Body:

        {
            "username": "newUser",
            "password": "password123",
            "email": "user@example.com",
            "birthday": "YYYY-MM-DD"
        }

Response:

        {
            "username": "newUser",
            "password": "hashed_password",
            "email": "user@example.com",
            "birthday": "YYYY-MM-DD"
        }

6. Allow users to update their user info

Method: PUT

URL: /users/:username

Description: This endpoint allows users to update their information such as username, password, email, and birthday.

Request Body:

        {
            "username": "updatedUser",
            "password": "newPassword",
            "email": "newemail@example.com",
            "birthday": "YYYY-MM-DD"
        }

Response:

        {
            "username": "updatedUser",
            "password": "hashed_new_password",
            "email": "newemail@example.com",
            "birthday": "YYYY-MM-DD"
        }

7. Allow users to add a movie to their list of favorites

Method: PUT

URL: /users/:username/movies/:MovieID

Description: This endpoint allows users to add a movie to their favorites list.

Request Params:

{ "username": "user123", "MovieID": "605c72d1f1c3c8bf4b5f678d" }

Response:

        {
            "username": "user123",
            "favorites": ["605c72d1f1c3c8bf4b5f678d"]
        }

8. Allow users to remove a movie from their list of favorites

Method: DELETE

URL: /users/:username/movies/:MovieID

Description: This endpoint allows users to remove a movie from their favorites list.

Request Params:

{ "username": "user123", "MovieID": "605c72d1f1c3c8bf4b5f678d" }

Response:

        {
            "username": "user123",
            "favorites": []
        }

9. Allow existing users to deregister

Method: DELETE

URL: /users/:username

Description: This endpoint allows existing users to deregister and delete their account.

Request Params:

{ "username": "user123" }

Response:

        {
            "message": "user123 was deleted."
        }
